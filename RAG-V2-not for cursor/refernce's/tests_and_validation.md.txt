## Shared Validation Helpers

Python-style validation helpers:

    def validate_query(query: str) -> None:
        if not query or not query.strip():
            raise ValueError("query cannot be empty.")


    def validate_top_k(top_k: int) -> None:
        if top_k <= 0:
            raise ValueError("top_k must be greater than zero.")


    def validate_candidate_pool_size(
        *,
        candidate_pool_size: int,
        top_k: int,
    ) -> None:
        if candidate_pool_size < top_k:
            raise ValueError("candidate_pool_size must be greater than or equal to top_k.")


    def validate_required_metadata(
        *,
        metadata: dict,
        required_keys: list[str],
    ) -> None:
        missing = [key for key in required_keys if key not in metadata]

        if missing:
            raise ValueError(f"Missing required metadata keys: {missing}")


    def validate_non_empty_string(
        *,
        value: str,
        field_name: str,
    ) -> None:
        if not value or not value.strip():
            raise ValueError(f"{field_name} cannot be empty.")

## Fake Embedding Client

Use fake embeddings for deterministic tests.

    class FakeEmbeddingClient:
        def embed_documents(self, texts: list[str]) -> list[list[float]]:
            if not texts:
                raise ValueError("texts cannot be empty.")

            return [self._embed(text) for text in texts]

        def embed_query(self, query: str) -> list[float]:
            if not query or not query.strip():
                raise ValueError("query cannot be empty.")

            return self._embed(query)

        def _embed(self, text: str) -> list[float]:
            normalized = text.lower()

            return [
                1.0 if "financial" in normalized else 0.0,
                1.0 if "revenue" in normalized else 0.0,
                1.0 if "password" in normalized else 0.0,
                1.0 if "contract" in normalized else 0.0,
            ]

Rules:

- fake embeddings should not call external APIs
- fake embeddings should make ranking assertions easy

## Fake Reranker

Use a fake reranker for deterministic reranking tests.

    class FakeReranker:
        def rerank(
            self,
            *,
            query: str,
            results: list,
            top_k: int,
        ) -> list[tuple[object, float]]:
            if not query or not query.strip():
                raise ValueError("query cannot be empty.")

            if top_k <= 0:
                raise ValueError("top_k must be greater than zero.")

            query_terms = set(query.lower().split())
            scored = []

            for result in results:
                text_terms = set(result.text.lower().split())
                score = float(len(query_terms & text_terms))
                scored.append((result, score))

            scored.sort(key=lambda item: (-item[1], item[0].chunk_id))

            return scored[:top_k]

Rules:

- reranking unit tests must be deterministic
- fake reranker must preserve predictable ordering
- tests must verify previous rank is preserved
- tests must verify score provenance is preserved

## End-to-End Hybrid Retrieval Test

A gold-standard end-to-end test should verify:

1. Documents load.
2. Text extracts.
3. Chunks are created.
4. Metadata is preserved.
5. Permission metadata is preserved.
6. Optional contextualization runs with fake generator.
7. Keyword index builds.
8. Embeddings are created using fake client.
9. Keyword retrieval returns ranked results.
10. Embedding retrieval returns ranked results.
11. RRF fuses both result lists.
12. Optional fake reranker reranks fused results.
13. Context builder returns final context.
14. Context respects token budget.
15. Final response includes results, sources, warnings, and config used.

Example flow:

    def test_end_to_end_hybrid_retrieval():
        document = make_test_document()

        chunks = chunk_text_by_paragraph(
            document,
            max_tokens=500,
        )

        keyword_index = build_keyword_index(chunks)

        embedding_client = FakeEmbeddingClient()

        embedded_chunks = embed_chunks(
            chunks=chunks,
            embedding_client=embedding_client,
            embedding_model="fake-model",
            provider_name="fake",
        )

        embedding_index = build_embedding_index(
            embedded_chunks=embedded_chunks,
            provider_name="fake",
            embedding_model="fake-model",
        )

        keyword_results = keyword_search(
            query="financial revenue",
            index=keyword_index,
            top_k=10,
        )

        embedding_results = embedding_search(
            query="financial revenue",
            index=embedding_index,
            embedding_client=embedding_client,
            top_k=10,
        )

        fused_results = reciprocal_rank_fusion(
            ranked_lists=[keyword_results, embedding_results],
            top_k=10,
        )

        context_result = build_context_block(
            fused_results[:5],
            config=ContextBuildConfig(max_context_tokens=3000),
        )

        assert context_result.context
        assert context_result.used_tokens <= 3000
        assert context_result.included_sources
        assert context_result.warnings is not None

## Unit vs Integration Tests

Unit tests:

- deterministic
- local
- fast
- no network
- no paid APIs
- use fake clients
- run on every change

Integration tests:

- may use real providers
- may require environment variables
- may require network
- may cost money
- should be explicitly marked
- should not run accidentally in normal unit test workflows

Integration tests should require explicit opt-in.

Example environment guard:

    import os
    import pytest


    pytestmark = pytest.mark.skipif(
        os.getenv("RUN_INTEGRATION_TESTS") != "true",
        reason="Integration tests are disabled by default.",
    )

## Bad vs Good

Bad:

    def test_rag():
        assert rag("hello")

Why bad:

- vague
- no expected behavior
- no metadata check
- no ranking check
- no token-budget check
- no permission check

Bad:

    def test_with_real_api_key():
        client = VoyageEmbeddingClient(model="voyage-3-large")
        assert client.embed_query("test")

Why bad for unit tests:

- requires network
- requires paid API
- can fail for reasons unrelated to logic
- may leak secrets if mishandled
- is not deterministic

Bad:

    def test_context():
        assert build_context(results)

Why bad:

- does not check token budget
- does not check source labels
- does not check warnings
- does not check skipped chunks
- does not check sensitive metadata leakage

Good:

    def test_metadata_survives_hybrid_pipeline():
        response = run_fake_hybrid_pipeline()

        assert response.results
        assert response.results[0].metadata["document_name"] == "Q4 Report"
        assert response.context
        assert response.included_sources

Good:

    def test_permission_filter_blocks_other_users():
        results = retrieve_context_for_query(
            query="financial revenue",

                filters={"user_id": "user_123"},
